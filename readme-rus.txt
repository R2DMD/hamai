PIPELINE: rx -> stt -> gpt -> tts -> tx
PROCESSED: stt -> gpt -> tts -> tx

rx.sh - Слушатель - ГОТОВ
1) Если шумодав открыт, пишет аудио поток в файл в директории pipeline/rx 
2) После закрытия шумодава, переносит аудиофайл в директорию pipeline/stt
3) Ждет нового открытия шумодава

stt.sh - Преобразователь аудио в текст (запускает stt.py) - ГОТОВ. Но нужно сравнить качество текста (может модель другую выбрать)
1) Ждет файлы в директории pipeline/stt
2) При появлении файла преобразует wav в txt
3) Сформированный текстовый файл сохраняет в директорию pipeline/gpt
4) Исходный аудиофайл переносит в директорию processed/stt

gpt.sh - Чат для общения (передача текста в LLM)
1) Ждет файлы в директории pipeline/gpt
2) При появлении файла передает текстовый файл в GPT
3) Ответ GPT сохраняет в файл в директории pipeline/tts
4) Исходный текстовый файл переносит в директорию processed/gpt

ttx.sh - Преобразователь текста в аудио - ГОТОВ
1) Ждет файлы в директории pipeline/tts
2) При появлении файла преобразует txt в wav c помощью piper
3) Сформированный wav-файл сохраняет в директорию pipeline/tx
4) Исходный текстовый файл перемещает в директорию processed/stt

tx.sh - Озвучиватель - ГОТОВ. Но не тестирован.
1) Ждет файлы в директории pipeline/tx
2) Про появлении файла воспроизводит через микрофон радиостанции
3) Исходный аудиофайл переносит в директорию processed/tx

start-pipeline.sh - запускает все скрипты для обработки pipeline

Варианты запуска:

ЕСЛИ все части pipeline на одной машине:
- то, запустить вручную каждый из 5 файлов .sh с этапами pipeline
- или запустить все их разом через start-pipeline.sh

Если части pipeline расположены на разных машинах, то разнести .sh файлы между машинами и обеспечить доступность директорий pipeline
